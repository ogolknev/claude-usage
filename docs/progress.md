# Claude Usage — состояние и обсуждение (на 2026-06-24)

Заметка-перенос контекста из рабочей сессии (сессия была запущена из `polygon`,
проект живёт здесь). Здесь: что сделано, что в релизах, и открытый вопрос про
встроенный логин.

## Что это
Menu-bar индикатор лимитов и расхода Claude для macOS (Rust). Кольцо-прогрессбар
сессионного (5ч) лимита + меню с недельным/модельным лимитом, временем сброса,
стоимостью и токенами. Источник лимитов — `GET /api/oauth/usage` с OAuth-токеном
Claude Code из Keychain; расход — из `~/.claude/projects/*.jsonl`.

## Готово и в релизах (v0.1.0 → v0.2.0)
- Кольцо в панели, цвет-градиент (HSV), процент в стиле батареи.
- Лимиты из `/api/oauth/usage` (client_id/endpoint/scheme вытащены из бинаря CC).
- Чтение токена через `/usr/bin/security` (один раз «Всегда разрешать», без
  повторных промптов) + фолбэк на `~/.claude/.credentials.json`.
- Локальный расход: дедуп по `message.id`, отдельно input+output и кэш; сверено
  с `ccusage` — совпадает.
- Стоимость по офф-тарифам Anthropic (cache 1ч ×2 учитывается); цены
  авто-обновляются из LiteLLM через jsdelivr в `prices.json`.
- Бэкофф при 429, кэш лимитов на диске (кольцо переживает рестарт/429).
- Дистрибуция: `.dmg` (`bundle/build-dmg.sh`), zip, Homebrew tap
  `ogolknev/homebrew-tap` (`brew install --cask ogolknev/tap/claude-usage`,
  cask снимает карантин), автозапуск `bundle/autostart.sh on`.
- Репозиторий: https://github.com/ogolknev/claude-usage

## Встроенный OAuth-логин (без Claude Code) — РАБОТАЕТ (с v0.2.0)
Логин в подписку прямо из приложения (для тех, у кого нет Claude Code). Детали —
в [oauth-login.md](oauth-login.md). Кратко: 6 дней `400 Invalid request format`
оказались **двумя багами**, найденными побайтовым диффом против успешного гранта
Claude Code:

- **`state` 16 байт** вместо 32 (CC шлёт 32) — бэкенд бракует короткий `state`.
  Фикс: `rand_bytes(16)` → `rand_bytes(32)`.
- **gzip token-ответа** — reqwest без фичи распаковки не читал тело. Фикс:
  features `gzip,brotli,deflate,zstd`.

Прежний вывод «блокер = Arkose» был **неверен**: ответ нёс `request_id` +
стандартный конверт API (валидация схемы, не WAF), а фейл был постоянным уже на
первой попытке (`rid` 1→71→38 непоследовательно). Arkose проходил прозрачно.

Токен теперь в **Keychain** (`security-framework`, `SecItem*`): наш `save()` пишет
туда, чтение — приоритет №3 (CC-токен перекрывает, наш = fallback), старый
плейнтекст-файл удаляется при сохранении. `refresh()` пишем тоже в Keychain.

### Решение по продукту
Логин включён в релиз (пункт меню «Войти в Claude…», флаг `--login`,
`--check` — диагностика). Оговорка: подписочный OAuth вне офиц. клиентов —
серая зона ToS.

## Возможные следующие шаги
- Получить успешный HAR Claude Code → дифф → вердикт по логину.
- Либо выпилить пункт «Войти в Claude…» из меню, чтобы не мозолил.
- Не связанное с логином — приложение готово, можно полировать (иконка app,
  пороги цветов кольца, CI-релизы).
