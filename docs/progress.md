# Claude Usage — состояние и обсуждение (на 2026-06-19)

Заметка-перенос контекста из рабочей сессии (сессия была запущена из `polygon`,
проект живёт здесь). Здесь: что сделано, что в релизах, и открытый вопрос про
встроенный логин.

## Что это
Menu-bar индикатор лимитов и расхода Claude для macOS (Rust). Кольцо-прогрессбар
сессионного (5ч) лимита + меню с недельным/модельным лимитом, временем сброса,
стоимостью и токенами. Источник лимитов — `GET /api/oauth/usage` с OAuth-токеном
Claude Code из Keychain; расход — из `~/.claude/projects/*.jsonl`.

## Готово и в релизах (v0.1.0 → v0.1.6)
- Кольцо в панели, цвет-градиент (HSV), процент в стиле батареи.
- Лимиты из `/api/oauth/usage` (client_id/endpoint/scheme вытащены из бинаря CC).
- Чтение токена через `/usr/bin/security` (один раз «Всегда разрешать», без
  повторных промптов) + фолбэк на `~/.claude/.credentials.json`.
- Локальный расход: дедуп по `message.id`, отдельно input+output и кэш; сверено
  с `ccusage` — совпадает.
- Стоимость по офф-тарифам Anthropic (cache 1ч ×2 учитывается); цены
  авто-обновляются из LiteLLM через jsdelivr в `prices.json`.
- Бэкофф при 429, кэш лимитов на диске (кольцо переживает рестарт/429).
- Дистрибуция: `.dmg` (`bundle/build-dmg.sh`), zip, Homebrew tap
  `ogolknev/homebrew-tap` (`brew install --cask ogolknev/tap/claude-usage`,
  cask снимает карантин), автозапуск `bundle/autostart.sh on`.
- Репозиторий: https://github.com/ogolknev/claude-usage

## Открытый вопрос: встроенный OAuth-логин (без Claude Code) — НЕ работает
Цель — логиниться в подписку прямо из приложения, чтобы работало у того, у кого
нет Claude Code (например, только Claude Desktop). Подробности и все находки —
в [oauth-login.md](oauth-login.md). Кратко:

- Аккаунт пользователя — личный **Claude Max 5x** (проверено по метаданным
  рабочего токена), значит подписочный claude.ai-флоу правильный.
- Наш грант **структурно корректен**: те же 5 scopes (без `org:create_api_key`,
  фронт его выкидывает), та же орг, полный набор полей.
- Грант (`POST claude.ai/v1/oauth/{org}/authorize`) всё равно отдаёт
  `400 invalid_request_error / Invalid request format`. В теле есть
  `arkose_session_token` (**Arkose/FunCaptcha**); поле `rid` растёт с попытками
  (1→71) — антибот сильно затриггерен с устройства.
- Claude Code в том же браузере (Zen) логинится — его вход так не затёрт.
- Версия бинаря CC — 2.1.170 (текущая). Оркестратор `startOAuthFlow`: поднимает
  локальный сервер, строит manual+localhost ссылки, открывает localhost в
  браузере и показывает manual как фолбэк. Оба варианта пробовали — падают.

### Точка, на которой остановились
Нужен **успешный** грант Claude Code (HAR, зелёный POST 200 на
`.../v1/oauth/<org>/authorize`) — побайтовый дифф с нашим упавшим поставит точку:
совпадёт (кроме arkose) → подтверждённый Arkose-тупик; отличие (напр. другой
`organization_uuid`) → это и есть фикс.

### Решение по продукту
Пока логин **в релизы не идёт** (антибот + серая зона ToS; долбить = риск
аккаунта). Код логина (`src/auth.rs`, пункт меню «Войти в Claude…», флаг
`--login`) остаётся в репо как задокументированный WIP/тупик.

## Возможные следующие шаги
- Получить успешный HAR Claude Code → дифф → вердикт по логину.
- Либо выпилить пункт «Войти в Claude…» из меню, чтобы не мозолил.
- Не связанное с логином — приложение готово, можно полировать (иконка app,
  пороги цветов кольца, CI-релизы).
